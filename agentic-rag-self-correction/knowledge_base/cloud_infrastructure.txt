COMMON AWS ISSUES AND RESOLUTIONS

ISSUE: EC2 Instance Unreachable
Symptoms: SSH timeout, application not responding, health checks failing
Resolution Steps:
1. Check instance status in AWS Console — verify running state
2. Review system logs via EC2 console (Actions > Monitor > Get System Log)
3. Check security group rules — verify port 22 or 443 is open from bastion
4. Verify NACL rules are not blocking traffic
5. Check if instance passed both status checks
6. If status checks failing — stop/start instance (not reboot) to move to new host
7. If EBS volume issue — detach and attach to recovery instance
Expected Resolution Time: 15-30 minutes
Escalation: cloud-ops@techcore.com

ISSUE: RDS Database Connection Failures
Symptoms: Application throwing connection timeout errors, database unreachable
Resolution Steps:
1. Check RDS instance status in AWS Console
2. Verify security group allows inbound 5432 from application servers
3. Check parameter group for max_connections setting
4. Review CloudWatch metrics for CPU, memory, IOPS
5. Check for ongoing maintenance window activity
6. Verify database credentials in Secrets Manager are current
7. Check read replica lag if using read replicas
Expected Resolution Time: 20-45 minutes
Escalation: dba-team@techcore.com

ISSUE: Lambda Function Failures
Symptoms: Function errors in CloudWatch, timeout errors, out of memory
Resolution Steps:
1. Check CloudWatch Logs for specific error messages
2. Review function timeout setting — increase if timing out
3. Check memory allocation — increase if out of memory errors
4. Verify IAM role has required permissions
5. Check VPC configuration if function needs private resource access
6. Review concurrent execution limits
7. Check dead letter queue for failed invocations
Expected Resolution Time: 15-30 minutes

ISSUE: S3 Access Denied Errors
Symptoms: Application cannot read or write to S3, 403 errors in logs
Resolution Steps:
1. Check bucket policy for explicit deny rules
2. Verify IAM role/user has s3:GetObject and s3:PutObject permissions
3. Check if bucket has public access blocked — verify intended
4. Review S3 access logs for specific request details
5. Check if KMS encryption key policy allows access
6. Verify correct AWS region endpoint being used
Expected Resolution Time: 15-30 minutes

COMMON GCP ISSUES AND RESOLUTIONS

ISSUE: Cloud Run Service Unavailable
Symptoms: 503 errors, service not responding, deployment failed
Resolution Steps:
1. Check Cloud Run service status in GCP Console
2. Review deployment logs for container startup errors
3. Verify container image exists in Artifact Registry
4. Check memory and CPU limits — increase if OOM errors
5. Review minimum instances setting for cold start issues
6. Check VPC connector if service needs private resource access
7. Verify service account has required permissions
Expected Resolution Time: 15-30 minutes
Escalation: cloud-ops@techcore.com

ISSUE: GKE Pod Crashes
Symptoms: Pods in CrashLoopBackOff, OOMKilled, service degraded
Resolution Steps:
1. kubectl describe pod <pod-name> for events and status
2. kubectl logs <pod-name> --previous for crash logs
3. Check resource requests and limits — increase memory if OOMKilled
4. Verify ConfigMaps and Secrets are correctly mounted
5. Check liveness and readiness probe configuration
6. Review node resource availability — kubectl describe nodes
7. Check for node pool autoscaling limits
Expected Resolution Time: 20-45 minutes
